## 액션 노드란
트리거가 '시작 신호'라면, 액션 노드는 '실제 작업자' 

Trigger Node → Action Node → Result

“언제 시작할지 → 무엇을 할지 → 어떤 결과를 낼지”

### 액션 노드의 3가지 특징
- 데이터 처리
  - 받은 데이터를 가공하고 변환하는 역할
  - 새로운 형태로 변환하거나 계산 수행
- 외부 연동
  - 다른 시스템이나 서비스와 소통하여 데이터를 주고받거나 외부 기능을 활용
- 결과 생성
  - 구체적인 결과물을 생산하는 역할을 함. 데이터를 기반으로 실제 산출물을 만듬. 

### 필수 액션 노드 7가지
- 데이터 처리
  - EDIT Fields
    - 데이터 변환의 기본, 필드 추가/수정/제거
  - Code
    - 무한 가능성의 커스텀 로직 구현 
  - AI Transform
    - 자연어 기반 변환 
- 외부 연동
  - HTTP Request
    - 외부 API 연동 및 웹 통신의 핵심
  - OpenAI
    - AI 기능 활용
    - GPT-4o 및 DALL-E 통합 
- 논리 & 제어
  - IF
    - 조건부 처리, 워크플로우 분기 처리
  - Data & Time
    - 날짜/시간 처리 및 계산의 전문가
